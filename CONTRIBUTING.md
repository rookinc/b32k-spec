# Contributing

Thanks for helping improve the Aletheos / B32k spec!

## Process
1. Open an issue describing your proposal.
2. Submit a PR targeting `main`.
3. Add entries to `CHANGELOG.md` under **Unreleased**.
4. CI will run `tools/validate_packet.py` on any JSON in `examples/`.

## Style
- Keep the spec succinct; put rationale into `docs/` if needed.
- Use **semver** for version bumps.

## Governance
- Editors: Scott A. Cave (Human Anchor); Gizmo/Aletheos (advisory); Community reviewers.
